## 4. 測試模式完整說明

### Mode A — 合成單塊測試（快，~5 分鐘）

```bash
cd ~/projects/【...】
./test/testbench/vit/build/tb_vit
```

- 不需要 weights 或真實影像
- 測試 **1 個 Transformer block** 的注意力（6 heads × HW FA）
- 驗證：多次 HW call 能否正確執行

**預期輸出**：
```
[TB/VIT] Mode A — Synthetic single-block test
...
[TB/VIT] *** SYNTHETIC TEST PASSED ***
```

---

### Mode B — 完整 12 層 HW 推論（慢，~1 小時）

```bash
./test/testbench/vit/build/tb_vit weights/
```

- 需要 `weights/` 目錄中有完整的 ViT-Small/16 預訓練權重
- 使用 **合成 sin/cos 影像**（不需真實圖片）
- 測試：72 次 HW FA call 能否全部正確完成

**預期輸出**：
```
FA calls total    : 72
Total FA cycles   : 53819784   ← 72 × 747497
Per-call cycles   : 747497
Predicted class   : 905  (合成影像，class 值無意義)
```

**背景執行方式**（避免等待）：
```bash
nohup ./test/testbench/vit/build/tb_vit weights/ > /tmp/mode_b.txt 2>&1 &
# 之後查看結果：
cat /tmp/mode_b.txt | tail -20
```

---

### Mode C — 真實影像單張推論

```bash
# SW 模式（快，秒級）
./test/testbench/vit/build/tb_vit weights/ /path/to/img.bin [class_id]

# 範例：測試 batch_50 的第一張（class 0 = tench）
./test/testbench/vit/build/tb_vit weights/ /tmp/batch_50/img_0000.bin 0
```

**影像格式**：float32 CHW [3,224,224]，ImageNet 正規化，`.bin` 格式
（由 `scripts/batch_preprocess.py` 生成）

**預期輸出**：
```
Predicted class   : 0
Top-5 predictions :
  #1  class=   0  prob=0.8xxx  ← 正確
  ...
[TB/VIT] *** CORRECT (top-1) ***
```

### Mode D — 批次準確率測試

```bash

```

**輸出格式**：

## 5. 建置旗標說明

| 旗標 | 用法 | 效果 |
|------|------|------|
| `STATS_DEBUG=1` | `make STATS_DEBUG=1` | 每次 HW call 後在 stderr 輸出 cycle 與累積值 |
| `SCALE_TYING=1` | `make SCALE_TYING=1` | 啟用 Q/K 共用量化 scale（ScaleTying） |
| `RMSNORM=1` | `make RMSNORM=1` | 以 RMSNorm 取代 LayerNorm（需對應權重） |

**注意**：切換旗標時必須 `make clean` 再 `make`，否則 timestamp 未變，不會重新編譯：

```bash
# 正確做法
make clean && make STATS_DEBUG=1

# 錯誤做法（不會重新編譯）
make STATS_DEBUG=1   ← 若 binary 已存在會直接跳過
```

## 8. 常見問題排除

### Q: `make` 指令沒有輸出，直接結束

**原因**：Makefile 的預設 target 是 `hw_build`，不是 `all`。
**解法**：
```bash
make all          # 明確指定
# 或
make clean && make  # 已修正的 Makefile 版本（all 排在第一）
```

### Q: `docker cp` 報錯 "Could not find the file"

**原因**：路徑使用 `/root/projects/` 而非 `/home/cherish62113/projects/`。
**解法**：確認容器使用者名稱（`cherish62113`），使用正確的絕對路徑。

### Q: `batch_preprocess.py` 報錯找不到圖片

**原因一**：路徑使用了 `...` 縮寫（字面上是三個點，不是實際路徑）。
**解法**：使用完整絕對路徑，例如 `C:\Users\User\Desktop\...`。

**原因二**：資料集是編號資料夾格式（`00000/`, `00001/`...），但腳本版本舊。
**解法**：使用更新版的 `batch_preprocess.py`（已支援自動偵測編號資料夾）。

### Q: Mode A/B HW 測試卡住不動

**原因**：`wait_for_irq()` 中 Phase 1 的無窮迴圈（`while(FA_interrupt)`），硬體 interrupt 信號永遠不會被清除。
**解法**：在 `fa_call_and_accumulate()` 中加入 `get_fa_hal()->reset()` 的方式清除（已修正，詳見 06_修改紀錄）。

### Q: Stats 顯示 Total cycles ≈ 748K 即使跑了 72 次 call

**原因**：calls 2-72 只跑了 8 個 cycle（interrupt 未清除，立刻返回）。
**解法**：同上，加入 pre-call reset（已修正）。
