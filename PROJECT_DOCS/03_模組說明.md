## 1. FlashAttention 硬體模組

### 1.1 頂層包裝器（flash_attn_wrapper）

```
flash_attn_wrapper
├── AXI4 Slave 介面  ← MMIO 暫存器（控制、地址、scale）
├── AXI4 Master 介面 ← DMA 讀寫 Host Memory
├── FA 控制器
│   ├── 外迴圈：Br-tile 索引（0..N/Br-1 = 0..13）
│   ├── 內迴圈：Bc-tile 索引（0..N/Bc-1 = 0..13）
│   └── K/V Ping-Pong 管理
├── 14×14 Systolic Array（GEMM1 & GEMM2）
├── Online Softmax 單元
├── O Accumulator [14, 64] INT32
└── FA_interrupt 輸出
```

### 1.2 MMIO 暫存器映射

| 偏移 | 名稱 | 說明 |
|------|------|------|
| 0x00 | FA_CONTROL | bit0=start，寫1觸發，寫0停止 |
| 0x04 | FA_SHAPE | [31:16]=N, [15:0]=d |
| 0x08 | FA_TILE | [15:0]=Br |
| 0x0C | FA_Q_ADDR | Q INT8 buffer 低32位地址 |
| 0x10 | FA_K_ADDR | K INT8 buffer 低32位地址 |
| 0x14 | FA_V_ADDR | V INT8 buffer 低32位地址 |
| 0x18 | FA_O_ADDR | O fp32 output 低32位地址 |
| 0x20 | FA_Q_SCALE | Q 量化 scale（fp32 bits） |
| 0x24 | FA_K_SCALE | K 量化 scale（fp32 bits） |
| 0x28 | FA_V_SCALE | V 量化 scale（fp32 bits） |

### 1.3 執行時序

```
Host                          Hardware
  │                              │
  ├── fa_set_shape(196, 64) ────►│ 設定 N, d
  ├── fa_set_tile(14) ──────────►│ 設定 Br
  ├── fa_set_q/k/v/o_addr() ───►│ 設定 DMA 地址
  ├── fa_set_q/k/v_scale() ────►│ 設定量化 scale
  ├── reset_runtime_info() ─────►│ （只清軟體計數器）
  ├── [★] get_fa_hal()->reset() ►│ ARESETn pulse → 清除 FA_interrupt
  ├── fa_start() ──────────────►│ start=1，FA 開始計算
  │                              │
  │   ◄── DMA read (Q) ─────────┤ 讀取 Q tile
  │   DMA read (K/V) ◄─────────┤ Ping-Pong 預取
  │   ...                        │ 14 × 14 次內迴圈
  │   ◄── DMA write (O) ────────┤ 寫回計算結果
  │                              │
  │   ◄── FA_interrupt=1 ───────┤ 計算完成
  ├── wait_for_irq() 返回        │
  └── fa_stop() ───────────────►│ start=0
```

## 2. HAL 層（FlashAttnHAL）

**檔案**：`src/hal/flash_attn_hal.cpp` / `flash_attn_hal.hpp`

### 2.1 主要介面

```cpp
class FlashAttnHAL {
public:
    void init();              // 初始化：reset + reset_runtime_info
    void reset();             // 硬體 reset（ARESETn=0→1）← ★修改後多次使用
    void final();             // 清理（FST trace 等）

    bool memory_set(uint32_t addr, uint32_t data);  // MMIO 寫
    bool memory_get(uint32_t addr, uint32_t& data); // MMIO 讀

    void wait_for_irq();      // 等待 FA_interrupt=1（with DMA 服務）
    void reset_runtime_info();// 清零 elapsed_cycle / memory_read/write

    runtime_info get_runtime_info() const;
};
```

### 2.2 地址轉換機制（vm_addr_h_）

```
Host 記憶體是 64 位地址，硬體 DMA 只傳 32 位地址（AXI）。
解決：vm_addr_h_ = HAL 物件地址的高 32 位。
      靜態 buffer（s_Q_h, s_K_h...）與 HAL 物件在同一 4GB 空間內。
      DMA 地址 = vm_addr_h_ | ARADDR_M（低 32 位）

static float s_Q_h[196 * 64];  ← 必須是 static，確保地址穩定
static float s_K_h[196 * 64];
static float s_V_h[196 * 64];
static float s_O_h[196 * 64];
```

### 2.3 wait_for_irq 邏輯

```cpp
// 原始邏輯（第一次 call 後 FA_interrupt 永遠為 HIGH）：
while (!device_->FA_interrupt) {    // call 2+ 立刻退出 → 8 cycle bug
    clock_step(...);
    handle_dma();
}

// ★ 修正方式：在 fa_call_and_accumulate() 中，每次 flash_attention()
//   前先呼叫 get_fa_hal()->reset()，將 FA_interrupt 清為 0，
//   再讓 wait_for_irq() 正常等待新的完成信號。
```

## 3. Runtime Flash Attention（runtime_flash_attn.cpp）

### 3.1 flash_attention() 函數流程

```
flash_attention(Q, K, V, O, N=196, d=64, Br=14)
│
├── compute_scale(Q, N*d) → q_scale = max|Q| / 127
├── compute_scale(K, N*d) → k_scale = max|K| / 127   [或 ScaleTying]
├── compute_scale(V, N*d) → v_scale = max|V| / 127
│
├── quantize_to_int8(Q → Q_int8)
├── quantize_to_int8(K → K_int8)
├── quantize_to_int8(V → V_int8)
│
├── fa_set_shape(196, 64)
├── fa_set_tile(14)
├── fa_set_q/k/v/o_addr(Q_int8, K_int8, V_int8, O)
├── fa_set_q/k/v_scale(q_scale, k_scale, v_scale)
│
├── reset_runtime_info()     ← 清零計數器
├── fa_start()               ← start=1，觸發 HW
├── wait_for_irq()           ← 等待完成 + DMA 服務
└── fa_stop()                ← start=0
```

### 3.2 ScaleTying 實作

```cpp
#ifdef FA_SCALE_TYING
    // 使用 Q/K 的最大值作為共用 scale
    float qk_scale = fmaxf(compute_scale(Q, len), compute_scale(K, len));
    float q_scale  = qk_scale;
    float k_scale  = qk_scale;
#else
    // 各自獨立 scale（預設）
    float q_scale = compute_scale(Q, len);
    float k_scale = compute_scale(K, len);
#endif
```

## 4. ViT 推論引擎（runtime_vit.cpp）

### 4.1 VitHWStats 結構

```cpp
typedef struct {
    uint64_t total_cycles;        // 所有 FA call 的累積 cycle 數
    uint64_t total_memory_read;   // 累積 DMA 讀取 bytes
    uint64_t total_memory_write;  // 累積 DMA 寫入 bytes
    uint32_t fa_call_count;       // FA 呼叫次數（應為 72）
    uint64_t last_call_cycles;    // ★ 最後一次呼叫的 cycle 數（基準值）
} VitHWStats;
// ★ 宣告為 volatile：防止編譯器以 -O2 省略讀取-修改-寫入序列
static volatile VitHWStats g_hw_stats;
```

**交叉驗證**：`total_cycles ≈ fa_call_count × last_call_cycles`

### 4.2 fa_call_and_accumulate()（關鍵函數）

```cpp
static void fa_call_and_accumulate(float* Q, float* K, float* V, float* O,
                                   uint32_t n, uint32_t d, uint32_t br) {
    // ★ 修改①：pre-call hardware reset 解決 multi-call bug
    get_fa_hal()->reset();

    flash_attention(Q, K, V, O, n, d, br);

    struct runtime_info ri = get_fa_hal()->get_runtime_info();

    // ★ 修改②：明確的 volatile 讀-修改-寫，防止 -O2 elision
    uint64_t cy  = g_hw_stats.total_cycles;
    g_hw_stats.total_cycles = cy + (uint64_t)ri.elapsed_cycle;
    // ... 同樣處理 memory_read, memory_write, fa_call_count
    g_hw_stats.last_call_cycles = (uint64_t)ri.elapsed_cycle;

#ifdef FA_STATS_DEBUG
    fprintf(stderr, "[FA-STATS] call %2u: ri.cycles=%-10llu  accum=%-10llu  ...\n",
            cnt+1, ri.elapsed_cycle, cy + ri.elapsed_cycle);
#endif
}
```

### 5.2 RMSNorm 實作

```cpp
void rms_norm(float* x, const float* gamma, int N, int d, float eps) {
    for (int i = 0; i < N; i++) {
        float* row = x + i * d;
        // 計算 RMS = sqrt(mean(x²) + eps)
        float sumsq = 0.0f;
        for (int j = 0; j < d; j++) sumsq += row[j] * row[j];
        float inv_rms = 1.0f / sqrtf(sumsq / (float)d + eps);
        // 正規化並乘以 gamma（無 mean subtraction、無 beta）
        for (int j = 0; j < d; j++) row[j] = row[j] * inv_rms * gamma[j];
    }
}
```

### 6.1 Mode 分派邏輯

```cpp
int main(int argc, char** argv) {
    if (argc == 1)
        return run_synthetic_test();                    // Mode A

    if (argc == 2)
        return run_real_inference(argv[1], nullptr, -1); // Mode B

    if (argc >= 4 && strcmp(argv[2], "--batch") == 0) {
        int use_hw = (argc >= 5 && !strcmp(argv[4], "--hw"));
        return run_batch_accuracy(argv[1], argv[3], use_hw); // Mode D
    }

    // Mode C：weights + image [+ class]
    int cls = (argc >= 4) ? atoi(argv[3]) : -1;

}
```
